## L-P0f — GRPO KL 缺口闭合

[grpo.py](../../../../lighttrain/builtin_plugins/trainers/grpo.py) 的 `fit()` 旧实现 `freeze_as_ref(...)` 后把 `ref_policy`
作局部变量丢弃（带 `# noqa: F841` + `TODO`），`_grpo_step` 从不把 ref log-probs 喂进 `ctx.extras`；
而 [GRPOLoss](../../../../lighttrain/builtin_plugins/losses/rl.py) 仅当 `beta_kl>0 且 ctx.extras 有 log_probs_ref` 才算 KL
→ **配了 `beta_kl>0` 的 GRPO，KL 惩罚永远是 0**（实测）。

修复（接好 ref log-probs，对齐 PPO 的字段生命周期；喂入逻辑为净新增）：

- **per-token ref 路径**：[ref_policy.py](../../../../lighttrain/builtin_plugins/rl/ref_policy.py) `ReferencePolicy.log_probs` 新增
  `per_token: bool=False`，复用其 `@torch.no_grad()`，返回 `(B,T)`——**gather `input_ids[:, 1:]`**（与 trainer 的
  `log_probs_new` 同 token 轨道，**非** labels）、`(B,T-1)→(B,T)` 前补一列 0，对齐 k3 估计器的逐位相减。
- **trainer 接线**：`__init__` 增 `self._ref_policy` 字段（直调 `_grpo_step` 不炸）；`fit()` 每次先重置 ref，
  按 **实际生效 loss 的 `beta_kl`**（新 helper `_effective_beta_kl`，解包 `LossOnlyObjective.loss_fn`、遵 `loss:` seam-wins）
  gate：仅 `beta_kl>0` 才 `freeze_as_ref`（`beta_kl=0` 默认路径不 deepcopy）；`_grpo_step` 每步先 `pop("log_probs_ref")`
  再在 `labels is not None` 分支注入。
- **loss 边界守卫（根治所有漏接线）**：`GRPOLoss(beta_kl>0)` 缺 `log_probs_ref` 时**改为 fail-loud**（不再静默 0），
  兜住 composite / 未来 loss wrapper / 直调漏注入的所有路径。
- **lora-base 暂 fail-loud**：`lora_base_as_ref=True` + `beta_kl>0` 在 `fit()` gate 与 `log_probs(per_token=True)` API 层
  均抛清晰错误（该路径需 eval/恢复 train 态 + lighttrain LoRA `.inner` adapter 转发适配，单卡难测）→ 转后续（见「新登记」）。

**行为变化**：配有效 `beta_kl>0` 的 GRPO，KL 从「静默 0」变真正生效；额外成本 = `fit()` 一次 deepcopy（显存）+
每个 inner step 一次 ref 前向（算力）。`ppo_epochs>1` 时 ref 前向有冗余（ref 冻结、跨 inner epoch 恒等；
GRPO 默认 `ppo_epochs=1` 成本相等）——**登记为后续优化点**（标准做法 rollout 阶段算一次存 buffer）。
`_grpo_step` 在 `model.train()` 态算 `log_probs_new`、deepcopy ref 是 `.eval()`，与现状一致，属可接受近似。
`beta_kl=0`（默认）：不建 ref、不注入、`kl_loss=0`，**逐值与修复前一致**（行为中性）。

回归测试（聚焦 trainer 接线，k3 数学已由 [tests/losses/test_rl.py](../../../../tests/losses/test_rl.py) 覆盖，不重证）：
[tests/trainers/test_grpo.py](../../../../tests/trainers/test_grpo.py) 新增 8 例（形状=`(B,T)`/首列 0、`_effective_beta_kl` 解包、
注入刻意不同 ref → `kl>0`、直调+beta>0+无 ref → fail-loud、stale 清理、缺 labels → fail-loud、`fit()` gate 建/不建 ref、
beta=0 行为中性）；[tests/rl/test_ref_policy.py](../../../../tests/rl/test_ref_policy.py) + [tests/test_rl_ref_policy.py](../../../../tests/test_rl_ref_policy.py)
新增 per-token API 单测（默认仍 `(B,)`；`per_token=True` 为 `(B,T)`、首列 0、gather input_ids、detached、lora-base 抛错）。

## L-P0e — `warn_unused_ignores` 闭合

[pyproject.toml](../../../../pyproject.toml) `[tool.mypy]` 开 `warn_unused_ignores = true`，清掉本机实测的 **12 处**历史多余
`# type: ignore`（v0.3.0 记的「10」是 **L-P0d** 本机 nightly vs CI CPU-torch 存根分裂下的旧计数；权威集以 CI types job 为准）：

- **import 类（5 处）**：`pyarrow` / `lm_eval` / `bitsandbytes` / `transformers`(×2) 的 inline ignore 删除，
  改由 `[tool.mypy.overrides] ignore_missing_imports` 覆盖——对「本机装了 → unused、CI 没装 → import-not-found」两态都稳
  （CI types 仅装 `.[dev]`，不含 optional extra）。
- **非 import 类（7 处）**：`union-attr`/`override`/`arg-type`/`assignment`/`attr-defined` 等 env-independent 的多余 ignore 直接删除。

## 新登记（转后续）

- **GRPO lora-base ref-KL**：`lora_base_as_ref=True` 的 KL 参考需 eval/恢复 train 态 + lighttrain LoRA `.inner` adapter
  转发适配，单卡难测。本版多入口 fail-loud，转后续专门 RL PR。
- **PPO `_ref_policy` 未消费 / `lora_base_as_ref` no-op**：[ppo.py](../../../../lighttrain/builtin_plugins/trainers/ppo.py)
  `freeze_as_ref` 后从不消费、knob 空转、deepcopy 浪费。**注意 `PPOSurrogateLoss` 用 old-vs-new approx-KL、本无 ref-KL 契约**
  （非 GRPO 式静默丢 KL）——登记为「unused ref 分配 / knob no-op / 语义待澄清」，勿强行给 PPO 接 ref-KL。
