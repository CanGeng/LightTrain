## 修复

- **🔴 lab/fork(真崩溃)**:`_copy_checkpoint` 对指向目录的符号链接误走 `shutil.rmtree`——`dst.is_dir()` 跟随 symlink,而 py3.12+ 的 `rmtree` 拒绝 symlink 并抛 `OSError`。改为先判 `is_symlink()`、symlink 一律 `unlink()`。
- **lineage cycle 检测**:`LineageRecorderCallback` 每 finalize 一个新产物,都因刚写的 `produced_by` 边(run→artifact,depth-1)误报 self-feeding cycle 警告。`cycle_check` 增 `exclude_direct_parent`,跳过 depth-1 直接父(且**不标 visited**,使经更深路径回到同一 run 的真多跳环仍能检出)。
- **lineage `resolve_ref('#<id>')`**:不校验节点存在即返回该 id,导致 `lineage invalidate/pin/tag #9999` 对幻影 id 静默 `exit 0`。现经 `get_node` 校验,缺失返回 `None`(命令 `exit 1`),与文档"Returns None when not found"一致。
- **diagnostics/index_page**:`hasattr(bus, "quarantined")` 守卫在 `try/except` **之外**,而 `hasattr` 只吞 `AttributeError`——`quarantined` property 抛其他异常会穿透。将访问移入 try。
- **rl/buffers `all_values`**:仅检查首个 episode 的 `values` 决定是否返回 `None`,首个无、后续有时静默丢弃其余。改为 `any(ep.values is not None ...)` 判定、缺的零填充(与同文件 `_collate` 一致)。
- **data/prepgraph/join `_default_namespace`**:下划线起头的名(如 `_v1`)首段为空、落到硬编码 `'aux'`。fallback 改为原名,保留有意义的命名空间。
- **死代码清理**:`models/peft/_common._fallback_base_spec` 不可达的 `elif isinstance(_, nn.Module)`(每个 `nn.Module` 必 `callable`,总先命中 callable 分支)删除;`diagnostics/oom_report._fmt_bytes` 循环后不可达的 `return` 改为显式 `unreachable` 守卫(满足 mypy 返回检查、`# pragma: no cover`)。
- **文案**:`crash_bundle` 写指标失败时警告"omits metrics_recent.jsonl"不实(`open("w")` 已建空文件),改为"may be empty or partial"。
