## 概览

**不改任何运行时逻辑 / 控制流 / 签名 / 注册名**。安全前提:注册靠装饰器 `@register("cat","name")`,
`import_all_components` 递归 walk——子包深度与注册无关,移动只改导入路径。行为保持铁证:注册表 dump 前后**逐字节一致**。

## 组织原则(语义分明的 4 条规则)

1. 核心有该域则**镜像其嵌套**(`builtin_plugins/<域>/<子>/` = 核心同路径)。
2. 纯插件类按其注册的 **registry category** 归到对应核心域(`callback`→`callbacks/`、`rl_backend`→`rl/`、`sweep_backend`→`lab/`)。
3. **高性能变体包**作为顶层命名兄弟保留(`quant/`、`layer_offload/`)。
4. 去掉不一致的 `_backends` 后缀(backend 就是其 category 的一个实现)。

## 扩展性约定(为后续"更多训练流程 / 高性能变体"预留)

- **新增实现**:按其 `@register` 的 category 落到镜像核心域的路径——trainer→`trainers/`、objective→`objectives/`、
  loss→`losses/`、rl 生成后端→`rl/backends/`、架构→`optim/architectures/`、prep 节点→`data/prepgraph/nodes/`、
  eval metric→`eval/metrics/`。注册无需接线(递归 walk 自动发现);全新 category 调 `register_category()`。
- **高性能变体**:作为**顶层命名兄弟包**(沿用 `quant/`/`layer_offload/`,如未来 `triton/`/`fused/`),其模块
  `@register` 与参考实现**相同的 category 名**(recipe 按名选用),由可选 extra 门控。**不**嵌进它加速的 category 目录
  (变体常跨 category、且按可选依赖整体跳过)。
