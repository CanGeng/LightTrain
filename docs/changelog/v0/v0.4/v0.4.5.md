# v0.4.5

> **分布式收窄到「仅数据并行」+ DeepSpeed ZeRO-2 真机验证 + 4 个沿途真 bug 修复**：删除张量 / 流水线 / 专家 / 序列并行（TP / PP / EP / SP）的全部骨架与接线，分布式只保留 **DDP / FSDP / DeepSpeed ZeRO** 三条数据并行路径；DDP、FSDP、DeepSpeed ZeRO-2 在真实单机 4×H20（NCCL）上跑通验证；修掉真机验证过程中暴露的 4 个缺陷。注册表 **35 → 33 category / 141 → 134 entries**（移除 `model_parallel_strategy`、`pipeline_schedule` 两类及其 7 个条目）。

## 概览

此前分布式层铺了一张「全并行谱系」的网：`ModelParallelStrategy`（TP / SP / EP）与 `PipelineSchedule`（1F1B / GPipe / Interleaved）两套 Protocol、对应的 registry category、`ParallelContext` 里的 tp/pp/ep rank·group·degree、`ParallelSection` 的 tp/pp/ep/sp/tensor_parallel/pipeline 字段，以及 runtime 里的 `_build_model_parallel_strategy` / `_build_pipeline_schedule` / `_preflight_parallel` 接线。但其中 SP/EP 从未真正接入 runtime（选择器只认 `tensor_parallel`，EP 是 skeleton），TP/PP 也从未真机验证。

本版做两件事：**砍**——把 TP/PP/EP/SP 整条线从代码与配置 schema 中移除，分布式收敛为「数据并行 only」；**证**——在真实 8 卡 H20 上把保留下来的 DDP / FSDP / DeepSpeed ZeRO-2 三条路径逐一跑通，并修掉验证途中暴露的 race / teardown / 后端 seam / tied-weight 四个真 bug。

全程保持 **`pytest` 4468 passed / 7 skipped、`mypy lighttrain tests` 零报告（573 文件）、`ruff` 净**。

- **两个 registry category 删除**：`model_parallel_strategy`、`pipeline_schedule`（`KNOWN_CATEGORIES`），及其 7 个内置条目 `tensor_parallel` / `tp_aware` / `sequence_parallel` / `expert_parallel` / `1f1b` / `gpipe` / `interleaved_1f1b`。`builtin_plugins/distributed/` 下的 `model_parallel/`、`pipeline/` 整目录删除。
- **`ParallelContext` 收窄到 dp-only**：去掉 `tp_rank/pp_rank/ep_rank`、`tp_degree/pp_degree/ep_degree`、`tp_group/pp_group/ep_group`、`sp_enabled`、`is_pp_last_stage` 及手动建组 helper（`_compute_ranks` / `_create_groups_manual` / `_create_ep_groups`）。`from_env` 改建 1 维 `(dp,)` DeviceMesh，校验从 `dp×tp×pp == world_size` 简化为 `dp == world_size`。
- **`ParallelSection` 收窄**：删 `tp` / `pp` / `ep` / `sp` / `tensor_parallel` / `pipeline` 字段及 `TPConfig` / `PipelineConfig` 两个子模型，只留 `backend` / `dp` / `force_cpu` / `grad_sync`。
- **runtime 接线删除**：`_build_model_parallel_strategy`、`_build_pipeline_schedule`、`_preflight_parallel` 移除；`_build_primary_model` 退化为「按 spec 解析模型」（不再做 TP 手术 / PP 切分）。
- **死代码清理（同 commit）**：删未被任何处引用的 `builtin_plugins/distributed/_launcher.py`（其 `validate_topology(dp,tp,pp,ep)` 与 dp-only 矛盾），及 `CALLBACK_EVENTS` 中 6 个从无发射点的 PP 事件名（`on_pipeline_schedule_*` / `on_microbatch_*` / `on_pipeline_bubble`）。
- **recipes 删除**：`recipes/3d_parallel.yaml`、`recipes/tp_ddp.yaml`。
- 净删除约 **−1.7K 行**（含测试树同步裁剪）。

## DeepSpeed ZeRO-2 启用 + 真机验证

- **`zero_grad` 提升为 `GradSyncStrategy` 协议方法**：把「DeepSpeed engine 没有 `set_to_none` kwarg、且 `engine.step()` 已清零」这一后端差异收敛进策略层（ZeRO 实现为 no-op，DDP/FSDP/noop 转发 `optimizer.zero_grad(set_to_none=True)`），调用点不再硬编码 `optimizer.zero_grad(set_to_none=True)`。
- **ZeRO `train_micro_batch_size_per_gpu`**：新增构造参数，让 DeepSpeed 从 live `world_size` 推导 `train_batch_size`（原先硬编码 `train_batch_size=1` 在 `world_size>1` 被拒）；单卡回落 `train_batch_size=1`。
- **ZeRO `clip_grad_norm` None 守卫**：`get_global_grad_norm()` 在首个 step 前返回 `None`，旧代码 `float(None)` 抛 `TypeError`；改为缺失时报 `0.0`（该值仅信息性）。
- **新增 recipe**：`recipes/nano_zero2.yaml`（tiny_lm + ZeRO-2 / NCCL）、`recipes/nano_fsdp.yaml`（tiny_lm + FSDP FULL_SHARD / NCCL），均带 console + jsonl logger。
- **验证范围**：DDP、FSDP、DeepSpeed ZeRO-2 在真实单机 4×H20（NCCL）跑通（loss 正常下降）；多机仍未验证。

## 沿途真 bug 修复

- **多卡 `run_dir` 各 rank 不一致（timestamp race）**。`make_run_dir` 用各 rank 本地 `datetime.now()` 打时间戳，跨整秒的 torchrun launch 会把同一次训练的各 rank 劈进相邻目录（checkpoint / metrics / lineage 全部分裂）。新增 `broadcast_run_dir`：先 init 进程组，**rank 0 创建目录并广播路径**，所有 rank 共用（`broadcast_object_list` 的 collective 同步点保证非主 rank 拿到路径时目录已就绪）。`setup_run_from_config` 内把拓扑初始化重排到 run-dir 创建之前。
- **lineage `database is locked`**。多 rank 同开一份 per-run SQLite 抢 WAL 锁。改为 **lineage store 仅 rank 0 开**（非主 rank 给 `None`，走既有软失败路径），既正确又消除锁竞争。
- **退出未销毁进程组**。多 rank `torchrun` 退出时未调 `destroy_process_group()`，每个 rank 刷资源泄漏告警。`train` / `resume` 的 `finally` 加 best-effort `_teardown_distributed()`。
- **tied-weight 诊断快照崩溃**。`safetensors.save_model` 拒存别名/共享 storage；ZeRO 把参数压进扁平 buffer 后，tiny_lm 绑定的 `tok_emb.weight` / `lm_head.weight` 成为互不覆盖的切片，导致 crash / frozen-step / nan-repro 诊断快照存模型时抛 `RuntimeError`。新增 `save_model_safe`：捕获后 clone state_dict 打破共享再 `save_file`（诊断快照不需保留 tying 元信息）。
- **recipe `mixed_precision` YAML bool**：`recipes/fsdp.yaml` / `recipes/zero2.yaml` 的 `mixed_precision: no` 被 YAML 解析为布尔 `False`，加引号改为 `"no"`。
