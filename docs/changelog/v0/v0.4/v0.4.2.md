# v0.4.2

> **新能力**:框架首个**图像监督训练**工作流(LM 起家的 lighttrain 此前几乎无图像训练)。
> 注册表 **138 → 141 entries**(新增 `model:tiny_cnn`、`loss:classification`、`collator:image`),35 category 不变。
> 同时一次行为保持的 **模态化** 结构调整:`models/` 与 `data/collators/` 引入模态二级轴。

## 概览

目标 2"扩展训练流程"的第一刀:给框架装上真正的图像监督能力——既补 LM-first 的根性缺口,又是
Kaggle"监督判别族"(图像/序列/token 分类、分割、音频分类)的地基。范式跑通(toy 规模),**复用通用
`pretrain` trainer**——无新 trainer / objective / update_rule。

结构上,把散落的 `model` 与分裂的 collator 按**模态**收拢:这是 `builtin_plugins/` 比核心更细的组织约定
(核心无此层),与 v0.4.1 的 category 主轴同构**不冲突**。移动部分注册表**逐项不变**;新增 3 项为真·新能力。

## 模态化迁移(旧 → 新,行为保持)

| 旧 `builtin_plugins/…` | 新 `builtin_plugins/…` | 依据 |
|---|---|---|
| `models/adapters/`(tiny_lm, hf_causal) | `models/text/` | models 模态二级轴;`adapters` 名不再贴切 |
| `data/core/collators.py`(CausalLM + Preference) | `data/collators/text.py` | collators 模态二级轴(镜像 `data/processors/` 先例) |
| —(新建) | `eval/metrics/`(docstring-only 空脚手架) | 未来 `@register("metric")` 落点,镜像核心 `eval/metrics` |

`data/core` 仍 re-export CausalLM/Preference Collator,**符号导入向后兼容**。**架构包不动**:
`optim/architectures/{mamba,rwkv,diffusion_unet,jepa}` 是 model+profile 的高内聚整体(profile 引用模型内部结构),
作为"自带 profile 的新架构"保留——`model` 类目因此有两个**语义明确**的家:`models/<模态>/`(应用模型)与
`optim/architectures/`(新架构整体)。

## 视觉 pilot 新增组件

- `models/vision/tiny_cnn.py` — `TinyCNNClassifier`(conv→自适应池化→线性头;分辨率无关)。
- `losses/classification.py` — `ClassificationLoss`(rank-2 CE + top-1 acc;`loss_family="classification"`)。
- `data/collators/image.py` — `ImageClassificationCollator`(`pixel_values (B,C,H,W)` + `labels (B,)`)。
- `examples/vision_components.py` — demo `synthetic_image` 数据集(类条件可分,零外部数据)。
- `recipes/image_cls.yaml` + `examples/image_cls.py`(用 `loss: classification`,复用 `trainer: pretrain`)。

## 组织约定(模态二级轴)

- **主轴 = category**(镜像核心,注册表/resolver 语义);**二级轴**:`models/` 与 `data/{collators,processors}/`
  按**模态**(text/vision/audio/multimodal),`losses/` 按 family,`objectives/`/`trainers/` 扁平按范式。
- 竖切的"按工作流"视图活在 `recipes/` + `examples/`,**不**进组件树。
- 新增判别类落点:模型→`models/<模态>/`、判别 loss→`losses/`、模态 collator→`data/collators/<模态>/`、
  注册式指标→`eval/metrics/`。

## 顺手修复
