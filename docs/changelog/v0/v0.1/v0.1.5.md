# v0.1.5

## 概览

**v0.1.5 是项目首次大规模对抗性测试基础设施补强。** 横跨四个独立维度——数学正确性、控制流正确性、存储/拓扑/缓存契约、解析/构造/运行时契约——共新增 **1148 个对抗性测试**，覆盖 32 个源码子模块。其中：

- **29 个 `test_regression_*` 回归 pin**：每个都有明确的 changelog 或既有 test docstring 引用，对应真实历史 bug，并通过 mutation sanity check 验证「源码回滚到 pre-fix 状态时测试必须 FAIL」。
- **3 处源码改动**：1 处真 bug 修复（`cli/_app.py` 的 patch flatten 列表退化问题），2 处 docstring 显式化（`EventBus` / `JSONLLogger` 的 main-thread-only 并发契约）。
- **3 处 known limitation**：测试过程中发现的契约-实现轻微不一致，本版本不修源码，记录于此供未来 PR 追溯。
- **1 个文档化 GAP**：`@pytest.mark.xfail(strict=True)` 标记的已知缺陷，写法为「正确行为」，源码修复后会 XPASS 强制移除标记。

全项目 **1858 测试通过 + 1 skipped + 1 xfailed**，全部既有遗留测试同时通过，无破坏。

## 动机

现有 `tests/test_*.py` 大量陷入「形状/接口/集合成员测试陷阱」：

- **算法层** (`tests/test_losses_*.py` 等)：只断言 key 存在、`tensor.shape`、`loss` 是 finite。若把 `torch.min(surr1, surr2)` 偷偷改成 `surr2`，或把 GRPO k3 KL 的 `Δ = log_ref - log_new` 写反，现有测试**全部仍然通过**。
- **控制流层** (`tests/test_engine_*.py` 等)：生命周期事件用 `event in fired`（集合成员）检查，**任意重排序都能通过**。三路 backward 分支只测裸路径，SKIP/RETRY/STOP 信号优先级从未被测，PPO value head 懒初始化从未被测。
- **存储/拓扑层** (`tests/test_checkpoint_*.py` 等)：只断言 `mgr.list_steps() == []`、`assert hits` 非空、`tensor.shape == (...)`。无 stale-pointer 状态、无 `dp×ep` 真实拓扑，没有解析解对比。
- **解析/构造/运行时层** (`tests/test_config.py` 等)：`tests/test_config.py:test_load_basic` 只断言 `cfg.mode=="pretrain"` 不覆盖 override 解析的任何边界；`tests/test_invariants_dsl.py` 覆盖 4 个 dunder bypass 但缺 chained subscript→attr / lambda / 类型描述符变体；`tests/test_realtime_control.py` 只测 `lr.json` 解析成功，不测 nan / inf / 0 / 负 scale 被拒绝；`tests/test_data_core.py` 只断言 `attention_mask.sum() == ...`，**不验证 dtype**、**不验证 4D `hidden_states_layers` 的 `(B,L,T,H) → (L,B,T,H)` 转置**。

本版本通过 **6 类对抗性测试手段**把契约钉到测试里：

1. **解析解对比**：构造输入使数学解可手算，断言 `torch.testing.assert_close(atol=1e-5, rtol=1e-4)`（Lion 第二步 `p_new=[0,0]`、EMA `m·pt + (1-m)·ps`、attention_mask `[1,1,1,0,0]`）
2. **极端边界与攻击载荷**：YAML 1.1 magic bool（`mode=on`）、注释字符（`tag=#x`）、unicode key、dunder 链（`loss.__class__.__mro__[1].__subclasses__()`）、subscript→attr 链、lambda 内 dunder、列表推导内 dunder、nan/inf scale、A=-1 ratio=2、Δ=0.5、ratio=2 vs exp
3. **严格顺序列表 + mock 探针**：用 `_OrderedRecorder` 断言 `events == expected_list`，事件重排序立刻失败；`_FakeGradSync` / `_FakeAccelerator` 记录每次调用，断言路由到正确分支；`fake_dist` 单进程内捕获 `dist.new_group` 调用列表，断言 EP/DP/TP/PP 组成员完全等于解析公式
4. **可观测信号 pin**：通过 `inspect.getsource` 检查 EventBus / JSONLLogger 源码不含 `threading.Lock` 且 class docstring 含 `"Thread-safety: NOT thread-safe"` 字面量；用 `id()` 比较 value head 跨 step 不被重建
5. **真实多进程**：`torch.multiprocessing.spawn` + gloo 真起 2–4 个进程，文件 rendezvous 避免端口竞争，验证 EP 组成员的真实 all_reduce 求和
6. **历史 bug 反向 pin + Mutation Sanity Check**：每个 `test_regression_*` 在源码暂时回滚到 pre-fix 状态时**必须 FAIL**

## 共享约定

1. **数值断言**一律用 `torch.testing.assert_close(actual, expected, atol=1e-5, rtol=1e-4)`
2. **禁止**用 `assert "key" in output` 或 `assert tensor.shape == (...)` 作为唯一断言
3. **docstring 必含**：测试目标 / 输入构造 / 解析解推导（或攻击路径）
4. **严格命名规范**：
   - `test_regression_<BUG_ID>` — 仅限真实历史 bug，必须能在 `docs/changelog/v0.1.*` 或既有 test docstring 找到引用（或本 PR 内发现+修复并显式标记），**不从 git log 推测**
   - `test_invariant_<contract>` — 永恒契约（原子性、写顺序、KL ≥ 0、cycle_check 不含 start_node 自身等）
   - `test_pin_<behavior>` — 当前可变行为（float 量化精度、shard 边界比较符、`re.search` 子串语义等），docstring 必须含警示语
5. **共用 fixtures**：分布式相关 `fake_dist_env` / `tiny_model` / `tiny_optimizer` / `tiny_batch` 在 `tests/conftest.py`；losses 专用 `seeded_rng` / `dummy_ctx` / `dummy_model_output` 在 `tests/losses/conftest.py`；distributed-specific `dist_mock` 在 `tests/distributed/conftest.py` 避免 namespace 冲突
6. **目录镜像**：`lighttrain/<module>/<file>.py` → `tests/<module>/test_<file>.py`
7. **新文件与现有扁平 `tests/test_*.py` 并存**，旧文件不动
8. **slow 测试**统一加 `@pytest.mark.slow`，`-m "not slow"` 可跳过

## 维度互不重叠的设计

四组联合覆盖了**算法数学**、**生命周期编排**、**存储/拓扑/缓存契约**、**解析/构造/运行时契约**四个独立失败维度，互不重叠：

- losses/rl/objectives 组的测试无法抓 EP 组成员错配、YAML magic bool、DSL dunder 绕过

- config/cli/registry/... 组的测试无法抓 KL 方向反、事件重排序、fingerprint None 碰撞、EP 组成员错配

每组各自定义"形状陷阱"的具体形态，并用对应的解析解/严格顺序/可观测信号/真实多进程/历史 pin 手段把契约钉死。彼此都不修改对方源码与测试。
