# v0.1.9

## 概览

**v0.1.9 实现 GaLore 实验（第二个「论文复现即用户反馈」实验，压测优化器协议 + 梯度面接缝）总结 §5 提出的 P0/P1/P2 全部改动，并修复 v0.1.8 用 `resume-verify` 发现但未修的 BUG-1。**

GaLore 实验的结论是：lighttrain 的优化器接缝**健全**（梯度投影优化器 + 跨步 SVD 状态、零核心改动、逐位精确 parity 与 resume），唯一 MAJOR 是 `estimate` 看不见 GaLore 的优化器状态节省（issue #4）。本版闭合该 MAJOR 及其余 6 个 MINOR，并把 BUG-1 从「发现」升级为「修复」。

- **修复 BUG-1（最高严重度开放正确性问题）**：默认采样器 mid-epoch resume 现在**逐步精确且 prefetch 无关**（`num_workers>0` 下也成立）。
- **新增 `optim_state_bytes` 优化器协议钩子**：`estimate` 现在能反映优化器的**真实**状态占用（内存高效优化器如 GaLore / Lion / 8-bit Adam 不再被当作全秩 Adam）。
- **param-group DSL 新增 `min_ndim` / `module_type` 谓词**：内置分组路径可直接表达「Linear 权重、ndim≥2」这类选层（如 GaLore），无需自定义 `build()`。
- **优化器-wrapper 契约文档化** + 修正 README「Custom optimizer」范例（旧范例照抄即在首个 `ckpt_every` 崩）。
- **`eval` 守卫** + 自定义状态 checkpoint **可移植化** + 第三方导入提示泛化。
- **无 BREAKING**：协议/DSL 改动均为向后兼容的加法；旧 checkpoint 仍可加载（缺 `batch_in_epoch` 时退化为 epoch 粒度 resume，不劣于旧版）。

## 🐞 BUG-1 修复 —— mid-epoch resume 逐步精确（prefetch 无关）

v0.1.8 用 `resume-verify` **发现**了 BUG-1（采样器只恢复到 epoch 粒度，mid-epoch resume 在边界后一步发散），但交付的是「发现该 Bug 的工具」而非修复。**v0.1.9 修复它。**

**根因**：`SequentialSampler` / `ShuffleSampler` 只存 `_epoch`，不记录 epoch 内已消费位置。

**修复设计（机制要点：消费计数源自 trainer，而非 sampler yield）**：

- 新增 `StepContext.batch_in_epoch`（[engine/_context.py](../../../../lighttrain/engine/_context.py)）——训练循环每次 `next(iterator)` 自增一次、epoch 滚动归零。它数的是 **trainer 实际消费的批**，**不是** sampler/prefetch 已吐出的批。
- **为什么这很关键**：`DataLoader` 在 `num_workers>0` 下预取，sampler 会比 trainer 多 yield `prefetch_factor × num_workers` 批。若用 sampler 的 yield 计数（旧 `stateful_resumable` 的潜在隐患），mid-epoch resume 在多 worker 下会**跳过过多**——只在 `num_workers=0` 的玩具配置下「碰巧」正确。源自 trainer 的计数则与 prefetch 深度无关。
- checkpoint 持久化 `batch_in_epoch`（[trainers/base.py](../../../../lighttrain/trainers/base.py)）；resume 时 `load_checkpoint` 调用 `data_module.seek(epoch, batch_in_epoch)`（[trainers/pretrain.py](../../../../lighttrain/trainers/pretrain.py)），把权威位置换算为索引偏移（`× batch_size`）写进采样器，**覆盖**任何 prefetch 偏斜的 yield-time 位置。
- 三个采样器都实现 `seek(epoch, consumed_indices)`：在按 `seed+epoch` **确定性重建**的本-epoch 顺序上跳过 `consumed_indices` 后继续。`stateful_resumable` 的消费账也对齐到权威计数（修其 yield-time 潜在隐患）；`num_workers=0` 下与旧值一致，故**不回退**其现有 bit-exact 路径（R7）。

**验收**（`resume-verify` 此前 mid-epoch FAIL，现 PASS）：

```
resume-verify: phase1=3 phase2=3 tol=0.01
  step 3: single=… resume=… Δ=0.000e+00   ← 保存点
  step 4: single=… resume=… Δ=0.000e+00   ← resume 边界（旧版此处 2.86e-02 FAIL）
  max |Δloss| = 0.000e+00  →  PASS
```

`tests/cli/test_resume_verify.py` 的 mid-epoch 用例已从「断言 FAIL」翻转为「断言 PASS」，并新增 **`num_workers∈{0,2}` 矩阵**（证明修复在多 worker 下也成立，而非只在单进程）。

> 闭环再走一步：v0.1.8 的工具产生分歧 → 定位 BUG-1 → v0.1.9 修复并用同一工具回归验证。

## 新增 / API

### `optim_state_bytes` 优化器协议钩子（issue #4，MAJOR）

`estimate` 此前按 `2 × trainable_param_bytes` 估算优化器状态，对 `galore_adamw` 与 `adamw` 报同一数字——GaLore 的全部卖点（内存）在框架自带估算器里隐形。

- `OptimizerWrapperProtocol` 新增**可选** `optim_state_bytes(model) -> int`（[protocols.py](../../../../lighttrain/protocols.py)）。
- `_BaseWrapper` 提供默认实现（Adam 类 `2×`、Lion `1×`），不需 `build()`（[optim/wrappers.py](../../../../lighttrain/optim/wrappers.py)）。
- `estimate` 经 registry 实例化 wrapper 并 `getattr` 调用钩子，缺失时回退旧 `2×params`（[lab/estimate.py](../../../../lighttrain/lab/estimate.py)）。**行为变化**：estimate 现会（a）导入 recipe 的 `user_modules`（使用户注册的优化器/模型可解析），（b）实例化优化器 wrapper（此前只读名字）；对内置优化器无副作用，钩子设计为不需 build。实测 GaLore：`optim_state_bytes` 由回退的 6.56 MB 变为 **2.31 MB**（−65%，与实测 checkpoint 逻辑状态 2.42 MB 相差 ~5%），adamw 基线仍 6.56 MB。
- GaLore adapter 覆写钩子：低秩 Adam 矩 + SVD 投影器字节数（[experiments/galore/registrations.py](../../../../experiments/galore/registrations.py)），选层逻辑与 `build()` 共用一个 `_select_galore_params`。

### Param-group DSL 谓词 `min_ndim` / `module_type`（issue #3）

`ParamGroupSpec` 在名字正则之后追加可选过滤：`min_ndim`（仅 ndim≥N，排除 1-D bias/norm）、`module_type`（按 `type(module).__name__` 匹配，如 `Linear`）。语义保持首匹配胜出。

```yaml
param_groups:
  - pattern: "attn|mlp"
    min_ndim: 2
    module_type: Linear
    rank: 32
```

## 硬化（failure-first，真实 GaLore 链路测试中暴露）

补齐 `galore_torch` 后跑通真实链路，暴露两个早先「框架级等价证明」无法发现的问题，已修复：

- **`estimate` 不再把「优化器无法 resolve」和「优化器没钩子」混为一谈**（[lab/estimate.py](../../../../lighttrain/lab/estimate.py)）。此前两者都**静默**回退 `2×params`——名字打错或 `user_modules` 导入失败时，estimate 会给出一个看似真实的错数字。现在：**resolve 失败 / 钩子抛错 → 显式 `UserWarning`**（提示检查名字与 `user_modules`），**仅缺钩子 → 静默回退**（合法，多数优化器本就如此）。
- **`_BaseWrapper._safe_state_dict(convert)` helper —— 把作者从手写 `state_dict` 的 aliasing 坑里捞出来**（[optim/wrappers.py](../../../../lighttrain/optim/wrappers.py)）。`torch` 的 `state_dict()` 返回的内层 `state[param]` dict 与活动优化器**同引用**；原地改写自定义状态（把 projector 换成纯张量）会**污染正在运行的优化器**，下一个 `.step()` 直接崩（实测 `AttributeError: 'dict' object has no attribute 'project'`）。helper 先 copy 内层 dict、再对每条目应用 `convert(key, value)`，安全。GaLore adapter 的 P2a `state_dict()` 已改用它（dogfood）；契约文档与 README 均加了 aliasing 警告。
