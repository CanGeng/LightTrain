# v0.2.4

## 概览

落地 `FIXDOCS.md` 文档审查报告：把文档与实现重新对齐（EN + ZH + 两个 README），并修复审查暴露的三类「配置看似生效、实则被静默忽略」代码缺陷，外加一处测试隔离修复。

- **文档**：纠正过期事实、中英文漂移、功能「说满」（EventBus 事件数、CLI flags、注册表类别、分布式状态、PrepGraph 语义、quickstart 等）。
- **代码（fail-loud）**：Optuna sweep 后端断链；TP/PP/SP/EP 配置无法生效时改为抛 `ConfigError`（不再静默回落单卡）；自动诊断 callback 构造失败不再静默吞。
- **测试**：补 fail-loud / sweep / 诊断单测 + 一条集成测试；修复一处 `.env` 引起的 `os.environ` 跨测试泄露 flake。

范围决策：文档 + 全部代码修复；EN + ZH + README 同步；并行无法生效一律 fail-loud。

## 代码修复

**Optuna / `sweep_backend` 断链**
- `KNOWN_CATEGORIES` 增加 `sweep_backend`（[registry/_core.py](../../../../lighttrain/registry/_core.py)），与 optuna 插件的 `@register("sweep_backend","optuna")` 对齐 → 实际类别数 34 → 35。
- [pyproject.toml](../../../../pyproject.toml)：空的 `optuna = []` 改为 `sweep = ["optuna>=3.0"]`（与错误/文档里的 `.[sweep]` 一致）。
- `_optuna_configs`（[lab/sweep.py](../../../../lighttrain/lab/sweep.py)）：lookup 前 `import_all_components()`（sweep CLI 不经 `load_config`，插件可能尚未注册）；捕获从 `(ImportError, KeyError)` 改为 `(ImportError, RegistryError)`，未装 optuna 时给出 `.[sweep]` 安装提示而非裸异常。包路径提示更正为 `lighttrain.plugins.sweep_backends.optuna`。CLI 错误打印加 `rich.markup.escape`，避免 `[sweep]` 被 Rich 当标记吃掉。

**并行配置 fail-loud（TP/PP/SP/EP）**
- `_build_model_parallel_strategy`（[cli/_runtime.py](../../../../lighttrain/cli/_runtime.py)）：修正判定顺序（`tp>1` 但缺 `tensor_parallel:` 块不再静默返回 `None`）；策略未注册 / `apply()` 失败 → `ConfigError`；新增 `parallel.sp` / `parallel.ep>1` 守卫（已注册但未接入 runtime → `ConfigError`，不再静默无效）。
- 抽出 `_build_pipeline_schedule(cfg)`：读取 `parallel.pipeline.schedule`（默认 `1f1b`，支持 `gpipe` / `interleaved_1f1b`），未知 schedule → `ConfigError`，并从构造 kwargs 中 `pop("schedule")`（`gpipe` ctor 无此参数）；`prepare()` 失败 → `ConfigError`。
- 并行配置 preflight 前移到创建 run dir 之前：纯配置错误不再污染 `runs/`，且精确 `ConfigError` 先于 `_init_parallel` 的 `RANK expected`。

**自动诊断 callback 不再静默失败**
- `_auto_attach_m4_callbacks`（[cli/_runtime.py](../../../../lighttrain/cli/_runtime.py)）：关键 `InvariantsCallback` 构造失败 → fail-loud（重抛 `ConfigError`）；非关键（`FrozenStepCallback` / `FileSignalsCallback` / `CallbackIsolationSink`）→ `warnings.warn` 后跳过（不再 `except: pass`）。

## 文档修复（EN + ZH + README）

- **EventBus**：事件数 39 → 46（10 处：README×2、architecture、extending、registry、diagnostics 的 EN/ZH）；失败策略措辞补「critical（`critical=True` 或默认名单）重抛终止 / 非关键连续失败后 quarantine」。
- **CLI**：`cli.zh-CN.md` 的 `convert-checkpoint`（`--from/--to/--path/--out`）、`export`（`--to/--ckpt/--out [-c]`）对齐实际签名；`train` 自动跑 PrepGraph 的条件收紧为「训练数据显式引用 prep_graph 输出」。
- **配置必填**：`model:/data:/optim:` 旧三段改为「模型声明（含 `models:`/`model_profiles:`）+ 数据声明 + 每个 trainable model 的 optimizer（含 `optimizers:`）」（configuration EN/ZH + getting-started）。
- **Quickstart**：`init` 不生成 `corpus.txt`，文案改为「先放一个 `corpus.txt` 再训练」（getting-started EN/ZH + 两个 README）。
- **Recipes**：补 `ddp`/`fsdp`/`zero2`/`tp_ddp`/`3d_parallel`/`nano_model`，新增「分布式 overlay」段并说明 overlay 需叠加完整 recipe + 多进程启动器（recipes EN/ZH）。
- **扩展文档**：中文 architectures 补「custom trainer objective-seam 契约」「custom engine 必须容忍 `loss_fn=None`」两节；多模型 trainer 机制补「显式 `models=`/`optimizers=` 或带 `**kwargs` 转发给 `Trainer.__init__`」。
- **注册表**：类别数 32 → 35，新增 `sweep_backend` 行 + `architecture` 小节，补中文漏掉的 sampler（`length_grouped`/`curriculum`/`stateful_resumable`）/ update_rule（`rl`/`forward_forward`/`pcn`/`dfa`）/ architecture（`transformer`/`rwkv`）。
- **分布式状态**：README / docs index / distributed 同步「SP/EP 已注册但未接入、请求即抛 `ConfigError`；EP 仍是 skeleton」，并补 fail-mode 清单。
- **如实降级**：diagnostics 自动挂载收紧为明确 4 项（`invariants`/`frozen_step`/`file_signals`/`CallbackIsolationSink`）；resume 降级为 functional resume（`PrepGraphDataModule` / `length_grouped` / `curriculum` 无 consumed-index seek）；架构页「rank-0 gates all IO」改为「checkpoint/logging 有 gate；run metadata / PrepGraph 未完全协调」。
- **其它**：PrepGraph fingerprint 公式补 `kind` / `schema_kind` / `SCHEMA_VERSION`，缓存路径区分独立 `prep`（`runs/<exp>/prep`）与训练内嵌（`run_dir/prep`）；PEFT 列表补 `ia3`；PackNode 脚手架 `max_len` → `seq_len`。

## 测试与隔离

- 新增（[tests/cli/test_runtime.py](../../../../tests/cli/test_runtime.py) / [tests/test_lab_sweep.py](../../../../tests/test_lab_sweep.py)）：TP 缺块 / 策略未注册 / `apply` 失败、SP / EP 守卫、pipeline schedule（`gpipe` 选中 + `pop("schedule")` / 未知 schedule / `pp≤1`）、自动诊断 fail-loud + warn + 空 config 正常构造、`sweep_backend ∈ KNOWN_CATEGORIES` + 缺依赖友好报错、一条经 `setup_run_from_config` 的集成测试（`tp:4` → `ConfigError` 且不建 run dir）。
