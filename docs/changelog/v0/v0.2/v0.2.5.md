# v0.2.5

## 概览

承 v0.2.2–v0.2.3 的 §3.3 纠偏**收尾到底**：把仍留在核心层的**全部具体 `@register` 实现**下放到
[lighttrain/builtin_plugins/](../../../../lighttrain/builtin_plugins)，使 **core 不再含任何已注册组件**——
只保留协议（[protocols.py](../../../../lighttrain/protocols.py)）、框架 seam（registry/config/engine 框架/
EventBus/PrepGraph 框架/logging 框架/eval 框架/invariants DSL）与无注册身份的 plumbing。原 research /
frontier 两层就此合并为单一 `builtin_plugins`。

分 4 批 relocate（**零功能变化**，仅三处刻意微改见下）；recipe 用短名解析，迁移对用户配置透明。

## 边界（三类归属）

- **(A) 抽象 / 框架 → 留 core**：`protocols` / `registry` / `config` / `engine/_context` /
  `trainers/base` / `callbacks/base`(EventBus) / `logging/_bus` / `architectures/profile` /
  `invariants/_dsl` / `eval` 框架 / `prepgraph` 框架 / `data/core/_schema` / `distributed/{_context,_protocols}` /
  `checkpoint` / `utils` / `minimal`。
- **(B) 已注册实现 → `builtin_plugins`**：判定 = `@register(...)` 装饰器**或**函数式 `register("cat","name")(Cls)`
  （后者如 `data/samplers`）。含 models·losses·optim·data 实现·prepgraph nodes·engine·各 trainer·
  rl·update_rules·architectures/transformer·callbacks·5 个注册型 diagnostics·invariants·realtime_control·
  logger 后端·eval `RegressionGate`·artifacts。
- **(C) 无注册身份的 plumbing → 按「谁消费」定夺**：被 core 框架/公开 API 消费的留 core
  （`data/{cache,mixing,packing}`、`trainers/{_primitives,_utils}`、`update_rules/_primitives`、
  `models/extras`·surgery、diagnostics 的 crash_bundle·oom_report·index_page·callback_isolation·
  frozen_step·nan_repro）；仅被搬走实现消费的随之下放（整个 `rl/`）。

## 三处非机械改动（其余皆纯 relocate + import 重定向）

1. **`_RowsDataset` 抽到 core**：从 `prepgraph/nodes/materialize.py` 拆出到
   [data/cache/_rows.py](../../../../lighttrain/data/cache/_rows.py)，使留-core 的 `PrepRunner` /
   `PrepGraphDataModule` 不再 import 已下放的 node 实现。
2. **`RegressionGate` 拆出**：从 `eval/suite.py` 拆到
   [builtin_plugins/eval/regression_gate.py](../../../../lighttrain/builtin_plugins/eval/regression_gate.py)，
   eval 框架（`Evaluator`/`EvalTask`/`EvalReport`/`metrics`）留 core。
3. **`default_objective` 去 CE**：[trainers/base.py](../../../../lighttrain/trainers/base.py) 的
   `Trainer.default_objective()` 改返回 `None`（core 不再识具体 loss）；CE 默认下沉到
   [builtin_plugins/trainers/pretrain.py](../../../../lighttrain/builtin_plugins/trainers/pretrain.py) 的
   `PretrainTrainer` override。补契约测试
   [tests/trainers/test_default_objective.py](../../../../tests/trainers/test_default_objective.py)
   （base→None、PretrainTrainer→CE；preference 缺 objective 响亮报错由 `test_gap_report_fixes` 覆盖）。

## 基类回归核心层（base classes → core）

§3.3 只下放**已注册的具体实现**；随实现一起被搬走的**未注册抽象基类**应留在 core。本轮把三个这样的
基类抽回核心层并改为公开命名（具体子类仍在 builtin_plugins，从 core 继承）：

- `_BaseWrapper` → [`OptimizerWrapperBase`](../../../../lighttrain/optim/base.py)（+ `ParamGroupSpec` 参数组 DSL）；
  `AdamWWrapper`/`LionWrapper` 继承之。
- `_SchedulerBase` → [`SchedulerBase`](../../../../lighttrain/optim/base.py)；4 个调度器继承之。
- `_BaseStore` → [`ArtifactStoreBase`](../../../../lighttrain/data/artifacts/base.py)（连同 `ArtifactHeader` / 错误类）；
  三个 store 继承之。顺手去掉 `store.py` 里与 [protocols.py](../../../../lighttrain/protocols.py) 重复的
  `ArtifactStoreProtocol` 定义，统一用核心协议。

新增两个**仅含抽象基类**的 core 包 `lighttrain/optim/`、`lighttrain/data/artifacts/`（计入 `_CORE_DIRS`）。
注册型 `PreferenceTrainer`（`trainer/preference`）/ `OneFOneBSchedule`（`pipeline_schedule/1f1b`）因带
`@register` 仍留 builtin_plugins；`PrepNode`/`Trainer` 等基类本就在 core，无需动。
