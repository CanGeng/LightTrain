# v0.5.4

> **nanoGPT + MiniMind 忠实移植 (PR2/2)**：在 `examples/` 顶层新增两个知名仓库的完整 lighttrain 本地化移植，分别展示「复用现有组件」与「扩展自定义架构」两条路径。

## nanoGPT — `examples/nanoGPT/`

[Andrej Karpathy / nanoGPT](https://github.com/karpathy/nanoGPT) 完整移植，架构与原版**逐位对齐**。

**新增文件**

| 文件 | 说明 |
|---|---|
| `model.py` | `@register("model", "nano_gpt")` — 完整 GPT-2 架构：可选 bias LayerNorm、`c_attn`/`c_proj` 命名（OpenAI Conv1D 兼容）、Flash Attention、weight tying、`from_pretrained("gpt2*")`（HF 权重 + Conv1D transposition）、`crop_block_size`、`estimate_mfu`、weight-decay 分组 AdamW |
| `components.py` | `@register("dataset", "memmap_bin")` `BinaryMemmapDataset`（uint16 .bin memmap）+ `@register("collator", "stack_xy")` `StackCollator` |
| `configs/train_shakespeare_char.yaml` | 6层字符级 (10.65M)，~5 min on GPU，预期 val loss ~1.47 |
| `configs/train_gpt2.yaml` | GPT-2 124M on OpenWebText，4天 A100 |
| `configs/finetune_shakespeare.yaml` | 从 `pretrained: gpt2` 微调 Shakespeare BPE |
| `data/shakespeare_char/prepare.py` | 字符级分词 → train.bin/val.bin (verbatim) |
| `data/shakespeare/prepare.py` | tiktoken BPE → train.bin/val.bin (verbatim) |
| `data/openwebtext/prepare.py` | OpenWebText 预处理 (verbatim) |
| `sample.py` | 独立生成脚本：从 lighttrain checkpoint capsule 或 `gpt2*` 权重加载，char-level / tiktoken 两套 encode/decode |

**与原版的差异**：训练由 `lighttrain train -c ...` 驱动（YAML recipe）而非 `train.py`；checkpoint 格式为 lighttrain capsule（`model.safetensors`）；DDP 通过 `torchrun ... -m lighttrain.cli train` 启动；`sample.py` 同时兼容 lighttrain capsule 和原版 `.pt` 格式。

## MiniMind — `examples/MiniMind/`

[jingyaogong/minimind](https://github.com/jingyaogong/minimind) 完整移植，覆盖 **7 条训练流 + 5 个推理/服务脚本**。

**架构**：RMSNorm、RoPE（YaRN 长文本扩展）、GQA（`num_key_value_heads < num_attention_heads`）、QK-Norm、SwiGLU FFN、MoE（可选）、KV-cache 推理、`GenerationMixin`；自定义分词器 vocab_size=6400（中文优化）。

**训练流映射**

| MiniMind 原始脚本 | lighttrain 配方 | 备注 |
|---|---|---|
| `train_pretrain.py` | `configs/pretrain.yaml` | `pretrain` trainer + `next_token` |
| `train_full_sft.py` | `configs/sft.yaml` | 同上，`minimind_sft` 数据集 |
| `train_lora.py` | `configs/lora.yaml` | `lora` model wrapper |
| `train_dpo.py` | `configs/dpo.yaml` | `preference` trainer + `dpo` loss |
| `train_grpo.py` | `configs/grpo.yaml` | `grpo` trainer |
| `train_ppo.py` | `configs/ppo.yaml` | `ppo` trainer |
| `train_distillation.py` | `configs/distillation.yaml` | `pretrain` + `distill` loss |

**新注册组件**（via `user_modules`）

| 名称 | 类型 | 说明 |
|---|---|---|
| `minimind` | model | `MiniMindForCausalLM` 的 lighttrain 封装；仅返回 logits，loss 由 objective 处理 |
| `hf_auto` | tokenizer | `AutoTokenizer.from_pretrained(path)` 通用 HF 分词器封装；可复用于其他 HF 模型 |
| `minimind_pretrain` | dataset | JSONL `{"text": "..."}` 预训练集 |
| `minimind_sft` | dataset | 多轮对话 JSONL SFT 集（含 chat_template + thinking tag 处理）|
| `minimind_dpo` | dataset | `chosen`/`rejected` DPO 集 |
| `minimind_rlaif` | dataset | RLAIF 提示集（GRPO/PPO 用）|
| `minimind_pad` | collator | 栈合已 pad 好的定长样本 |

**推理/服务脚本**（`scripts/`，适配 sys.path；无 lighttrain 依赖）：`chat_api.py`、`web_demo.py`（Streamlit）、`serve_openai_api.py`（FastAPI OpenAI 兼容）、`eval_toolcall.py`、`convert_model.py`（转 HuggingFace 格式）。

**MoE 注意**：`use_moe=true` 时模型产生 `aux_loss`；lighttrain 标准 `next_token` objective 不含 aux_loss，需自定义 callback/objective 叠加（见 KNOWN_ISSUES E3）。

## 框架层面的发现

**E3（新增 KNOWN_ISSUES）**：lighttrain 内置 tokenizer 仅 `byte`，HF AutoTokenizer 需用户在 `user_modules` 自注册。本次移植在 `model_adapter.py` 中注册了 `hf_auto` tokenizer 作为绕过方案；后续应内置。
