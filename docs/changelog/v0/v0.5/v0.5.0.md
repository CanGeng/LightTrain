## RL reference-KL（A1 / A2，新增能力）

- **A1 — PPO 接入 reference-KL**。此前 [PPOTrainer](../../../../lighttrain/builtin_plugins/trainers/ppo.py) 在 `fit()` 无条件 `freeze_as_ref()` 建参考策略，却全程无人消费，[PPOSurrogateLoss](../../../../lighttrain/builtin_plugins/losses/rl.py) 也无 `beta_kl`/KL 项 → ref 是死重、PPO 实际不施加 ref-KL。照搬 GRPO 已验证模板：`PPOSurrogateLoss` 加 `beta_kl` + k3 KL 项（缺 `log_probs_ref` 时 fail-loud）；`PPOTrainer` 加 `beta_kl` 构造参数（默认 0.0，对既有配置加性无影响），`fit()` 仅当 effective loss `beta_kl>0` 才建 ref，`_ppo_step` 注入 per-token `log_probs_ref`。
- **A2 — GRPO/PPO `lora_base_as_ref` 接入 per-token KL**。此前 `lora_base_as_ref=True` + `beta_kl>0` 在两个 trainer 与 [ReferencePolicy](../../../../lighttrain/builtin_plugins/rl/ref_policy.py) 都直接 fail-loud（adapter toggling 已实现但只支持序列级）。新增 `_lora_base_log_probs_per_token`（disable→per-token 前向→finally enable），去掉拒绝守卫，trainer 注入时传 `live_model`，删除 fit() 里的 raise。

## Checkpoint crash-atomic（C1）

[CheckpointManager.save()](../../../../lighttrain/engine/checkpoint/manager.py) 此前原地覆写 `step_<n>/` 下固定文件名，同 step 覆写中途崩溃会留混合文件集、毁掉上一份本来完好的 checkpoint。改为 **staging 目录 + 原子 swap**：全部文件 + manifest 先写进 `step_<n>.tmp.<pid>.<token>/`，再原子 `os.replace` 换入（已存在则先把旧副本改名挪开、换入新副本、再删旧；换入失败则还原旧副本再 raise）。`_STEP_RE` 不匹配 `.tmp`/`.old` 兄弟 → list/load/prune 自动忽略半成品（崩溃残留的孤儿目录惰性无害，不主动清扫以保并发同 run_dir 写入不互相删 staging）。DoD：`step_<n>/` 只会是完整已提交目录或不存在，绝不半覆写。
