# v0.5.1

> **mypy 类型债清零（B1 隔离区 12 → 0）+ check_untyped_defs ratchet 启动（B2）**：把 v0.5.0 显式留作 CI-verified 后续的 `ignore_errors` 隔离区(torch-stub 批 + 中难批)一次清空,并启用 `check_untyped_defs` 覆盖整个 `lighttrain/` 生产代码。纯类型改动、零运行时行为变化(新增的 `assert ... is not None` 仅复述既有不变量)。注册表 **33 category / 134 entries** 不变。

## env-parity 方法论(本轮关键)

torch-stub 批之所以在 v0.5.0 被 defer,是因为本机 nightly-torch 与 CI 的 CPU-torch 对 `nn.Module` 属性访问的类型推导分叉(`Tensor | Module`),**本机绿 ≠ CI 绿**(见 [v0.3.1](../v0.3/v0.3.1.md) 告诫)。本轮建了一个 **CPU-torch parity venv** 精确复现 CI types job 的安装序列(`pip install torch --index-url .../cpu` + `pip install -e ".[dev]" -c constraints.txt`,torch 2.12.1+cpu / mypy 2.1.0),以它为唯一权威逐模块清理,CI 仅作最终确认。所有修复均为 **env-invariant**(parity venv 与本机 nightly 双绿)。

## B1 — mypy 隔离区清零(`ignore_errors` 12 → 0)

删除整个 `[[tool.mypy.overrides]] ignore_errors=true` 隔离块,12 个模块全部真修到零(parity venv 下 95 → 0):

- **torch-stub 批**(architectures [diffusion_unet](../../../../lighttrain/builtin_plugins/optim/architectures/diffusion_unet/__init__.py) / [jepa](../../../../lighttrain/builtin_plugins/optim/architectures/jepa.py) / [mamba](../../../../lighttrain/builtin_plugins/optim/architectures/mamba/__init__.py) / [rwkv](../../../../lighttrain/builtin_plugins/optim/architectures/rwkv/__init__.py);distributed [ddp](../../../../lighttrain/builtin_plugins/distributed/strategies/ddp.py) / [fsdp](../../../../lighttrain/builtin_plugins/distributed/strategies/fsdp.py) / [zero](../../../../lighttrain/builtin_plugins/distributed/strategies/zero.py))。核心是同一行在两个 torch 版本报不同 error code(nightly `attr-defined`、CPU-torch `operator`/`return-value`),`warn_unused_ignores` 下任何**带码 ignore 都在某一环境失配**。统一改为 env-invariant 真修:wrapped-module / DeepSpeed-engine 专有方法用 `cast(Any, model).method(...)`;层迭代与 `embedding_layer_fn`/`head_layer_fn` lambda 用 `cast(Any, m).attr`;有状态架构的 `self._state: list | None` 经 if/else 后补 `assert ... is not None`。
- **中难批**(trainers [_preference_base](../../../../lighttrain/builtin_plugins/trainers/_preference_base.py) / [grpo](../../../../lighttrain/builtin_plugins/trainers/grpo.py) / [ppo](../../../../lighttrain/builtin_plugins/trainers/ppo.py),data [joined_dataset](../../../../lighttrain/builtin_plugins/data/artifacts/joined_dataset.py) / [producer](../../../../lighttrain/builtin_plugins/data/artifacts/producer.py))。`self.model: Any | None` 在消费前补 `assert self.model is not None`(narrowing 跨方法内调用保持);rollout 入参 `prompt_ids`/`reward_fn` 的 `Any | None` 同款 assert;移除一批纯一级类的 `# type: ignore[override]`(mypy 2.1.0 下不再判 override 冲突)。artifact store 的 `header`/`root`/`finalize`(在具体 [`ArtifactStoreBase`](../../../../lighttrain/data/artifacts/base.py) / 子类上,不在窄的读协议 `ArtifactStoreProtocol` 上)用 `cast(ArtifactStoreBase, store)` / `cast(Any, store)` 定点取真类型。

## B2 — `check_untyped_defs` ratchet 启动

`[tool.mypy]` 启用 `check_untyped_defs = true`。整个 `lighttrain/` 生产代码(含本轮 un-quarantine 的 12 个模块)在未注解函数体被检查后 **0 新错**;唯一噪声(358 个未注解 body 错)全在 `tests/`,通过新增 `[[tool.mypy.overrides]] module=["tests.*"] check_untyped_defs=false` 收缩 opt-out 暂避。这把生产代码永久纳入 check_untyped_defs,`tests/` 余量作为下一个 ratchet 目标(逐测试模块清,清完即删 opt-out)。见 [KNOWN_ISSUES.md](../../KNOWN_ISSUES.md) 的 B2 条目。
