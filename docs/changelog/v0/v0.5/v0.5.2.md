## 范围与方法

`check_untyped_defs=true` 自 v0.5.1 起检查未注解函数体,生产代码 0 错,但 `tests/` 有 358 个未注解 body 错经 `tests.*` opt-out 暂避。本轮按**注解优先**逐文件清空,以 CPU-torch parity venv(= CI types 视角)为权威。修复手段分布:

- **真实注解**(annotation-first):给推导过窄的局部/字面量补类型——测试样本 list 注解 `list[Mapping[str, Any]]`(collator 入参,list 不变性 + 字面量窄推导)、空 dict / 空 list 字面量补 `dict[str, Any]` / `list[dict[str, Any]]`、`var-annotated` 站点补注解、test fake 属性(如 `_FakeTokenizer.chat_template: str | None`)补注解。
- **`assert ... is not None`**(47 处):测试已知非 None 但类型为 Optional 的访问——trainer `self.model` / value-head、`Path | None` 路径在 `ZipFile`/`read_text` 前、`dict | None` / `Tensor | None` 索引前、`EvalReport | None` 等。仅复述既有不变量,零行为变化。
- **定点 `# type: ignore[code]`**(仅不可注解处):
  - `method-assign`(49 处)—— `obj.method = spy` / `torch.Tensor.backward = …` 等 monkeypatch,本质不可注解;
  - 故意 wrong-type 测试 —— coercion 测试(`batch_size="4"`、`eos_id=2.0`、`do_sample=1` 等测构造器强转)、duck-type(`encode(bytes)`)、validation(`mode="bogus"`、invalid kwargs);
  - 测试替身 —— `_NonMainPctx` / `_BrokenStore` / `_Ctx` 等 duck-typed double 赋给具体类型槽、`SimpleNamespace` 充当 config、lm_eval 模块 monkeypatch;
  - 动态 config 字段 —— `RootConfig | DictConfig` 上的实验/自定义字段访问(DictConfig 运行时动态存在);
  - 捕获 `-> None` 返回值断言(`func-returns-value`,如 `assert rule.setup(...) is None`)。

## 顺带的生产注解修正(行为保持)

[LengthGroupedSampler](../../../../lighttrain/builtin_plugins/data/samplers/length_grouped.py) 与 [CurriculumSampler](../../../../lighttrain/builtin_plugins/data/samplers/curriculum.py) 的 `__iter__` 返回注解由 `Iterable[int]` 改为 `Iterator[int]`(body 本就 `return iter(...)`,旧注解过宽)——生产从不 `iter(self)` 故未暴露,测试的 `iter(sampler)` / `list(sampler)` 触发;一处修正清掉 26 个测试错。
