# v0.3.2

> broad-except **静默化清零**：把全仓所有「广义静默吞噬」改为可观测，**严格行为不变**。
> 计划：`.claude/plans/docs-changelog-known-issues-limitations-drifting-lovelace.md`。

## 概览

起因：v0.3.1 排查 CI flake `test_train_emits_frozen_step_then_replay_runs` 时，真异常被
[frozen_step](../../lighttrain/diagnostics/frozen_step.py) 的 `except Exception: pass` **静默吞掉**，
只剩「空 zip 断言」、看不到真因（该模块 docstring 自承 "The cost is silent"）。

本版把全仓**广义静默吞噬**（`except Exception` / `except BaseException` 且 handler 内无
日志 / 告警 / re-raise）统一改为在吞噬前记一行 `_log.warning(..., exc_info=True)`——
**保留 swallow，不收窄异常类型，不删 `# noqa: BLE001`**，控制流 / 返回值 / 是否抛出**全部不变**，
唯一新增是日志输出。对齐既有先例（[base.py](../../lighttrain/trainers/base.py) `write_oom_report`、
[grpo.py](../../lighttrain/builtin_plugins/trainers/grpo.py) 次级异常）。

## 改动

- **112 处**广义静默 except 加 `_log.warning(exc_info=True)`，覆盖 **52 个文件**（每个文件补
  `import logging` + `_log = logging.getLogger(__name__)`，命名对齐项目约定）。子系统遍及
  `diagnostics`（frozen_step / crash_bundle / callback_isolation / oom_report / index_page）、
  `cli`（_runtime + commands/*）、`lab` / `lineage`、`builtin_plugins`（layer_offload / artifacts /
  callbacks / diagnostics / update_rules / realtime_control / logging_backends / peft / sweep_backends /
  judges / invariants / distributed）、`config._loader` / `prepgraph` / `utils` 等。
- **re-raise 处不动**：捕获后 `raise` / `raise SomeError(...)` 的 handler 本就 loud（异常向上传播、
  CLI 已 `console.print`），在其前加日志冗余且误导——逐站点甄别后**保留原样**（cli 一带尤多）。
- **窄类型预期回退不动**：`except json.JSONDecodeError` / `ImportError` / `FileNotFoundError` 等处理
  已知条件、非隐藏 bug，不在本轮范围。

### frozen_step 生命周期 guard 去静默（except 之外的第二层）

去 except 静默后发现 [test_cli_freeze_replay](../../tests/test_cli_freeze_replay.py) 的 CI flake 走的是
**`if ...: return` 静默 guard**（非 except），命中即不产 bundle 也无日志。本版把
[frozen_step 回调](../../lighttrain/builtin_plugins/callbacks/builtins/frozen_step.py) 的生命周期 guard 也变可观测
（行为不变，仅加日志、hot-loop 每原因只警告一次）：`on_train_start` 无 run_dir → 警告 writer 全程禁用；
`on_step_begin` batch 非 dict / `ctx.model|optimizer` 缺失 → 警告本 run 跳过快照；`on_step_end` commit 返回 None
（无快照）→ 警告；[FrozenStepWriter.commit](../../lighttrain/diagnostics/frozen_step.py) `snapshot is None` → 警告。
并把该测试改为**自诊断**：失败信息拼入 frozen_step 的 WARNING 日志、run 目录全树、train stdout，使一次 CI 重跑即定位确切 guard。

## 行为变化

无功能性行为变化：best-effort 路径仍 swallow、仍不崩训练；**仅多了 WARNING 级日志（带 traceback）**。
本版起因的 frozen_step 失败场景，重跑即可在日志里直接看到被吞的真实异常。

**hot-loop 日志登记（后续优化点）**：少数站点在逐 step / 逐 layer / 逐 metric 循环内
（`update_rules/standard.py` RETRY_STEP、`realtime_control/file_signals.py` 每步轮询、
`layer_offload/_engine.py`、`lineage_recorder` 逐 metric）。持续失败时 warning 每轮一条会刷屏——
本版按「最大可观测」统一 warning，**如噪音过多，后续可加 module 级 'already-warned' sentinel 去重**
（仍 warning 级、控制流不变）。

## 与 L-P3a 的关系（勿混淆）

L-P3a「裸 `# noqa: BLE001` 广义 except」的 **ratchet 仍在**（except 仍是广义、noqa 全部保留）——
本版只消除其「**静默**」属性，未收窄异常面、未动 ruff 配置。二者正交。

## 回归测试

[tests/test_broad_except_logging.py](../../tests/test_broad_except_logging.py) 新增 2 例锁「不再静默」契约：
- **frozen_step**：monkeypatch `_save_model` 抛错 → `commit()` 仍返回 None、磁盘无 bundle（行为不变），
  但 `caplog` 含带 `exc_info` 的 WARNING。
- **config `_leaf_exists`**：`${nonexistent}` 插值在 descend 时抛错 → 仍返回 False（行为不变）、`caplog` 有 WARNING。
