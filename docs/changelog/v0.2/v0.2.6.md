# v0.2.6

## 概览

清理「历次修复遗留的 known issues」+ 4 个新发现的静默正确性 bug，共 **6 项**，全部单卡可测、各带回归测试。
另有 3 项（SP/EP 接入、SAM `grad_sync`）读源码后判定本机单卡无法验证、且 EP 是无 MoE 宿主、无真 all-to-all 的
skeleton——强行接入会回退 v0.2.3 的 fail-loud 安全性，故本轮**不动、保留登记**（见下「仍开放」）。

## 修复

### 1. AMP + 多次 forward 崩溃（standard / sam / mezo update rules）

`accelerator.autocast()` 返回一次性的 `_GeneratorContextManager`，三个 update rule 都把它**缓存一次**再在多次
forward 里重复 `with` 进入，第二次进入精确抛
`AttributeError: '_GeneratorContextManager' object has no attribute 'args'`（accelerate 1.13.0 实测）。
standard 仅 `RETRY_STEP` replay 触发；**SAM（两遍）/ MeZO（L±）一旦开 AMP 就每个 step 必崩**。

- 新增共享 helper [`make_autocast(accelerator)`](../../lighttrain/engine/update_rules/_primitives.py)：每次 forward 新建 autocast CM。
- [standard.py](../../lighttrain/builtin_plugins/update_rules/standard.py) / [sam.py](../../lighttrain/builtin_plugins/update_rules/sam.py) /
  [mezo.py](../../lighttrain/builtin_plugins/update_rules/mezo.py) 改为在 forward 闭包内 `with make_autocast(accelerator):`，删掉缓存块。
- `_primitives.py` 模块 docstring 拓宽为「shared, re-entrant update-rule training primitives」，`__all__` 加 `make_autocast`。
- 回归 [tests/update_rules/test_amp_multi_forward.py](../../tests/update_rules/test_amp_multi_forward.py)：用一个 `autocast()` 返回**真·一次性** `@contextmanager` CM 的
  fake accelerator（conftest 的 `nullcontext` 桩可重入、恰好掩盖了此 bug）；覆盖 standard 的 RETRY 二次 forward、SAM/MeZO 双 forward。

### 2. checkpoint `load()` 读到 stale 文件（manifest-driven load，防御性硬化）

[manager.py](../../lighttrain/engine/checkpoint/manager.py) 的 `load()` 旧实现按**文件是否存在**加载 optimizer/scheduler/rng/trainer/data_module，
而非按 `manifest["files"]`；同目录二次保存若组件变少（旧文件残留）会被误读。改为**严格按 `manifest["files"]`**（manifest 即唯一真相）。

- **定性**：经核实 base trainer 的 step→best 都走同一 `_collect_state()`、optimizer 纳入对称，**不触发**此 bug；仓库内无「同目录省 optimizer 重存」路径。
  故本项是**「manifest 即唯一真相」的防御性契约硬化 + 合成复现**，非已确认的 live 生产 bug。
- `save()` 不改目录结构、不做孤儿清理（忽略孤儿就是稳定边界）。
- 同 step 覆写的**崩溃态**非原子（抽象局限）：仅在 `CheckpointManager.save` docstring 中性记一笔（短语 `not crash-atomic` / `same-step overwrite`），列为后续 PR 决策点。
- 回归 [tests/test_checkpoint_full_state.py](../../tests/test_checkpoint_full_state.py)：合成构造同目录异构 manifest（先带后不带 optimizer）→ `load()` 不读 stale optimizer；+ docstring-pin 短语。

### 3. 文本 chunk 配置静默丢 token（fail-loud）

[datasets.py](../../lighttrain/builtin_plugins/data/core/datasets.py) `LineFileTextDataset` 按 `chunk_size` 步进取片再 `[: max_len]` 截断，
`chunk_size > max_len` 时丢中间 token（实测 100 token / chunk_size=50 / max_len=20 只覆盖 40/100）。
改 line 39 赋值 `None if chunk_size is None else int(chunk_size)`（让 0/负数进入 guard），并 fail-loud 拒绝 `chunk_size <= 0` 与 `chunk_size > max_len`。
（把 0 由「静默关闭分块」改为 fail-loud 是有意收紧；无 recipe 传 0，`pretrain_rwkv.yaml` chunk_size=16/max_len=64 不受影响。）
回归 [tests/test_data_core.py](../../tests/test_data_core.py)：超界 / 0 / 负数抛 ValueError；`<= max_len` 全 token 覆盖；`None` 仍单行模式。

### 4. CrossEntropyLoss 对 2D logits 静默错位

[losses/core.py](../../lighttrain/builtin_plugins/losses/core.py) 的 next-token shift 守卫 `logits.dim() >= 2 and size(-2) == labels.size(-1)`
对 `(B,V)` 分类头 + `(B,)` labels 也成立（B==B 巧合），错误做了 batch 维 shift（实测 2.286 vs 正确 2.710）。
改为 `dim() >= 3`（仅 `(..., T, V)` 序列才 shift，保留任意前导维支持，只排除有歧义的 2D）。docstring 点明语义，`loss_family` 仍 `"next_token"`（不暗示完整分类 objective）。
回归 [tests/losses/test_core.py](../../tests/losses/test_core.py)：`(B,V)+(B,)` 结果 == `F.cross_entropy`（不 shift）。

### L1. ParquetRowStore 构造不 reload（闭合 v0.1.5 Known limitation L1）

[store.py](../../lighttrain/builtin_plugins/artifacts/store.py) `ParquetRowStore.__init__` 旧实现永远空起，跨进程 `open_artifact_store` 拿到空 store。
现 finalized 时从磁盘 reload（参照 `MemmapFixedStore.__init__` 的 `sample_to_row` 模式）：

- `_index` 以 `manifest.json["sample_to_row"]` 为 source of truth，读 `rows.parquet` 重建 `_rows` 并对行号 / `sample_id` 一致性 cross-check。
- **fail-loud**（不像 safetensors/memmap 那样吞 JSONDecodeError 变空）：manifest 缺失/JSON 损坏即 raise；`sample_to_row` 必须存在且是 dict（缺失即视为损坏 raise，不当空 store）；空 index 仅当 `count==0` 且无 `rows.parquet` 才算合法空 store，否则 raise；`sample_to_row` 非空却缺 parquet 即 raise。
- 剥 union-schema 的 None 列**按整三元组判定**：`<f>`/`<f>__shape`/`<f>__dtype` 全 None → 整组剔除；部分 None（损坏行）→ fail-loud。
- 补 `_finalized` 写保护（finalize 后 `put` 抛 `RuntimeError`），并**对齐到 MemmapFixedStore**，消除「三后端三语义」。
- 回归 [tests/artifacts/test_store.py](../../tests/artifacts/test_store.py)：`parquet-rows` 纳入 `_BACKENDS_WITH_RELOAD`；新增异构字段 reopen（验剥 null 列）、损坏行 fail-loud、post-finalize put 抛错（参数化全后端）；
  清理 4 处 stale 测试文档/死代码（`_skip_if_no_reload` 死函数删除等）。

### L3. CheckpointManager 单写者并发崩溃（闭合 v0.1.5 Known limitation L3 的崩溃面）

[manager.py](../../lighttrain/engine/checkpoint/manager.py)：两进程写同一 `last.json.tmp`，一方 `os.replace` 抛 `FileNotFoundError`。
把三个原子写 helper 的 tmp 名改为含唯一 marker 的中缀（`.tmp.<pid>.<token>`，新增 `import secrets`），消除崩溃。
**仍是 single-writer 契约**（「后写者胜」排序不保证），docstring 保留 `"only rank-0"`。
回归 [tests/checkpoint/test_atomic.py](../../tests/checkpoint/test_atomic.py)：两进程同 run_dir 不同 step → 不抛 FileNotFoundError、`last.json` 指向有 manifest 的 step；
两处 `.endswith(".tmp")` 断言改 `.tmp.` marker；中性化 race/exitcode-1 旧注释。
