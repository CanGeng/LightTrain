# v0.2.1

## 概览

修复一轮审计（[docs/fixme/](../fixme/)）暴露的「config 格式漂移只迁了 train 路径」缺陷：v0.1.8 的
`model:` 字符串选择器与 Step-4 的 `models:`/`optimizers:` 集，只有 `train` 端到端迁移；
`estimate` / `produce-artifact` / `export` / `dry-run --build` 各自留着旧解析器，在现代 recipe 上
要么静默走错、要么直接炸——`export` 在**每个** v0.1.8+ recipe 上都坏（把 `model:` 当 dict 解析）。

本次把**所有模型构建路径收敛到单一来源** [`config/_models.py`](../../lighttrain/config/_models.py)，并把
**组件注册从手维护清单换成 `load_config` chokepoint 上的自动发现**——从结构上消除「忘注册 / 忘迁移」这一
反复发作的 bug 类。三道 CI 锁把过去靠人工检查的不变量变成机器强制。

全程 bit-check 守护：单模型路径逐 tensor 一致。测试套件 **2022 passed / 0 failed**（基线 2004，+18）。

依据：[audit-known-bugs.md](../fixme/audit-known-bugs.md)、[models-build-paths.md](../fixme/models-build-paths.md)。

## 模型构建路径收敛（HIGH）

新建 [config/_models.py](../../lighttrain/config/_models.py)，作为模型/优化器声明归一的唯一来源：

- `normalize_model_set(cfg)` —— 任意声明形式（`model:`+`model_profiles:` 糖，或 `models:`/`optimizers:` 集）
  归一为 `(models_cfg, optimizers_cfg)`；同时接受 `RootConfig` 与 dict（estimate 公共 API 路径）。
- `primary_trainable(models_cfg)` —— 「首个可训练 + 无可训练错误」的**唯一 owner**（`setup_run_from_config`
  与 `build_primary_model` 共用，不再各自重抄那段选择逻辑）。
- `build_primary_model(cfg) -> (model, n_trainable)` —— dry-run / export / produce 的唯一构建入口。
- `optim_spec_for(entry, optimizers_cfg)` —— 优化器名回退解析（显式名→查表；None→main/首个）。

收敛后：

- **`export`** 删除 bespoke `_build_model_for_export`（把字符串 `model:` 当 dict 解析，v0.1.8+ 必崩），
  改走 `build_primary_model`；多可训练模型时打印响亮提示（导出的是被 checkpoint 的主模型，而非静默二选一）。
- **`estimate`** 删除自带 `_build_model` 与二次 `select_model_spec`，**只归一一次**，模型名 / 优化器取自同一
  主条目；现支持 `models:` 集（原先直接 raise）。
- **`dry-run --build`** / **`produce-artifact`** 经统一 `_build_model` 自动支持所有声明形式。
- 命名优化器若未声明，`normalize_model_set` 按名报错（含可用键），不再拖到构建期才给泛化错误。

不变量锁 [test_no_rogue_model_parser.py](../../tests/cli/test_no_rogue_model_parser.py)：声明解析
（`select_model_spec` / 裸 `cfg.model`·`cfg.models`）只许出现在 `config/_models.py` + `config/_resolver.py`——
第 5 个 rogue parser 无法静默发版。

## 组件注册：手清单 → 自动发现 + chokepoint

删除 `cli/_runtime.py` 中 ~90 行手维护的逐模块 import 清单（它静默漂移：`info_nce` / `moe_balance` 及
另外多项已注册组件从未被 eager 导入；`export` 还忘了调用它）。

新建 [config/_components.py](../../lighttrain/config/_components.py)：`import_all_components` 对一份 **curated 顶层
包清单**用 `pkgutil.walk_packages` **递归**发现并导入全部子模块（registrable 散在 `data.core.*`、`models.*`
等两级深处，单层枚举会漏）。失败契约 per-module：内部 `lighttrain.*` 导入失败 / 非 ImportError **响亮上抛**，
仅缺第三方可选依赖或 `plugins` 整包缺失才静默跳过（比旧的盲 `try/except` 更 failure-first）。

并入 [load_config](../../lighttrain/config/_loader.py) chokepoint：新增独立开关 `register_components`
（与 `import_user_modules` 正交），内建组件**先于** user_modules 导入。于是任何吃 recipe 的命令自动获得完整
注册表，`export` 无需再记得手动触发；`--print-config` 与无 `--build` 的 `dry-run` 走
`register_components=False` 保持纯 dump、不拖 torch-heavy 模块。

curated 清单覆盖**全部**含 `@register` 的顶层包，注册不依赖任何传递 import（在干净子进程中验证自给）。

安全网 [test_registry_superset.py](../../tests/cli/test_registry_superset.py)：在**干净子进程**里跑 `import_all_components`，
其产出的注册表对「换机制前 dump 的基线」逐类别**超集**（允许多、禁止少）——静默丢失注册、或注册退化为依赖
传递 import，都会 FAIL。契约锁 [test_components.py](../../tests/config/test_components.py)：注入假模块验证响亮 / 静默行为。

## 打包修复：插件收进 `lighttrain.plugins` 命名空间

发现一个真实分发 bug：`pyproject` 的 `[tool.setuptools.packages.find]` `include` 写的是不存在的
`frontier_plugins*`，而真实目录是顶层 `plugins/`。后果是**构建 wheel 时整个 `plugins/` 从不被打包**——
插件（layer_offload / qlora / vllm / 异构架构 / 分布式策略…）此前只在 editable/dev 模式下靠 cwd 在 sys.path 才能 import。

修法（避免最小补丁把极通用的顶层包名 `plugins` 钉进 wheel、与他库/用户本地 `plugins` 相互遮蔽→插件静默消失）：
把 `plugins/` 收进 **`lighttrain/plugins/`**，随 `lighttrain*` 自动打包、零顶层名冲突。

- `import plugins.X` → `import lighttrain.plugins.X`（核心 `lab/estimate`、examples、~12 个插件测试、插件内部自引用、recipe 注释）。
- [config/_components.py](../../lighttrain/config/_components.py) 的可选包从 `("plugins",)` → `("lighttrain.plugins",)`；
  插件内部真坏（`lighttrain.plugins.*` ImportError / 非 ImportError）现按 failure-first 契约**响亮上抛**，缺第三方依赖仍静默跳过。
- [pyproject.toml](../../pyproject.toml) `include` → `["lighttrain*"]`（删掉指向不存在目录的 `frontier_plugins*`）。
- 原 `plugins/distributed/recipes/` 的 6 个分布式 recipe 移到**顶层 `recipes/`**；
  [test_all_recipes_loadable.py](../../tests/config/test_all_recipes_loadable.py) 不再 glob 旧路径。
- 干净子进程超集网确认插件仍以 `lighttrain.plugins.*` 注册、零丢失（`cpu_offload`/`vllm`/`tiny_rwkv`/`qlora`/`forward_forward`…）。
