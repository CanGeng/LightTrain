## 概览

给装配层瘦身 + 立质量门禁。P0 先立**零容忍质量门禁**（ruff + mypy 入 CI）作为安全网，

## P0 — 质量门禁清零并接入 CI（已完成）

- **ruff**：`pyproject.toml` 新增 `[tool.ruff]`，`select = F/E/W/I/UP/B`，`ignore = E501`；
  Typer 假阳性用 `flake8-bugbear.extend-immutable-calls = [typer.Option, typer.Argument]`
  精确豁免（不豁免 B006）。`ruff check .` **全仓零违规**（autofix 818 处 + 手工 ~50 处）。
- **mypy**：新增 `[tool.mypy]`（py312，非 strict）。门禁基准 = 完整 typed 环境（CI types job 装 CPU torch）。
  无 stub 第三方按模块 `ignore_missing_imports`（accelerate/deepspeed/matplotlib/optuna/vllm/yaml）。
  `mypy lighttrain` **256 文件零报告**。
- **CI**：`.github/workflows/test.yaml` 新增 `lint`（ruff）、`types`（mypy + CPU torch）两个 job，
  `test` job 改用 `--index-url .../cpu`；三 job 共享 `constraints.txt`，`cache-dependency-path` 含之。
- **验证**：`ruff check .` 全绿、`mypy lighttrain` 全绿、`pytest` 2088 passed / 8 skipped / 0 failed。
  代码改动经 333 文件 import 重排 + UP 近代化 + 手工修复，行为保持（全量测试通过）。

## P2 — 拆 `setup_run_from_config`（已完成）

- **巨函数解体**：在 `cli/_runtime.py` **内部**把 ~438 行的 `setup_run_from_config` 抽成 13 个模块级
  阶段函数（`_resolve_config` / `_preflight_parallel` / `_prepare_run_dir` / `_open_lineage_store` /
  `_require_optim_spec` / `_build_primary_model` / `_build_trainable_core` / `_build_aux_models` /
  `_build_update_rule` / `_build_accelerator_from_cfg` / `_build_engine` / `_build_trainer` /
  `_wire_trainer_context`），主函数收敛为薄编排器。**不跨文件搬**——所有符号仍在 `_runtime` 命名空间，
  `monkeypatch.setattr(_runtime, "_registry_get", ...)` 等测试照常绑定。
- **不变量显式化**：`_preflight_parallel` 作为 `_prepare_run_dir` **之前**的独立阶段，保住「TP/PP 校验
  先于建 run_dir 副作用」（`test_runtime.py` 的「invalid parallel config 不建 run dir」用例通过）。
- **返回契约护栏**：新增 exact key-set 断言测试，覆盖**两条返回路径**——正常 18-键 bundle 与
  `print_config_only=True` 的早返回 `{cfg, resolved_yaml}`（`tests/cli/test_runtime.py`）。
- **DoD：`cli._runtime` 摘出 mypy 隔离**。基线 20 个错误已清零：15 个 `RootConfig|DictConfig` vs 窄形参
  由 `_resolve_config` 内一次 `assert isinstance(cfg, RootConfig)`（setup 始终 validate=True）统一收窄；
  另 5 个 helper 修复（`_load_state_dict_into` 的 `state: Any`；FrozenStep 重名 `cb→fs_cb`；
  `rt.enabled`/`rt.poll_every` 加 `rt is not None` 收窄；`pool_kind` 标 type:ignore）。
- **验证**：`ruff` 全清、`mypy` **269 文件**全绿、`pytest` 2090 passed、`dry-run` + 2 步 `train` 冒烟跑通。

## 已知问题（Known limitations）

> 沿用 L 编号约定；后续版本逐条「闭合」。

- **L-P0a — mypy 类型债 ratchet 隔离（50 模块）**：P0 基线 181 个 mypy 错误。为达成「gate 全绿」，
  对携带历史类型债的模块设 `[tool.mypy] ignore_errors = true`（见 `pyproject.toml`，带 TODO）。
  **非永久**——逐模块清理后从列表删除。**P1 已闭合 `cli._app`、P2 已闭合 `cli._runtime`**（52→50）；
  其余为 ratchet backlog（owner: lighttrain maintainers）。干净模块（~219 个）受完整 mypy 保护。
- **L-P0b — 行长 / 格式未进 gate**：`E501`（411 行 >88 字符）被 `ignore`；`ruff format` 未启用
  （全仓一次性格式化 diff 过大，待单独决策）。
- **L-P0c — constraints 仅锁工具链，非完整 transitive lock**：`constraints.txt` 精确 pin `ruff`/`mypy`
  （lint/types「无代码变更变红」的真正风险源）。**未做完整 transitive 锁**：本地 dev env 是定制 nightly
  （`torch 2.12.0+cu132` / CUDA 13、conda `@ file://` 包、git-editable `mamba_ssm`），无法在 CI 的公共
  CPU index 复现。CI 现装 CPU torch。
- **L-P0d — types job 版本漂移 / cpu-cuda 存根分裂**：因 L-P0c，CI 的 torch 公共版本与本地 nightly 不同，
  其 `.pyi` 存根可能有差异 → 某 clean 模块可能在 CI 报 mypy 错而本地不报。**暴露面已被 L-P0a 的隔离大幅缩小**
  （torch 重度模块都已隔离），但残留风险登记于此。
- **L-P0e — `warn_unused_ignores` 暂关**：开启后会报 10 处历史多余的 `# type: ignore`（位于非隔离模块）。
  作为独立 hygiene ratchet，待隔离清空后再开。
- **L-P0f — `grpo.fit` 的 `ref_policy` 计算后未消费**（`builtin_plugins/trainers/grpo.py`）：
  P0 以 `# noqa: F841` + TODO 保留（行为中性），疑为 GRPO KL 参考的潜在缺口，P3 调查。
